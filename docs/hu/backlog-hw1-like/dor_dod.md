
# Definições de Pronto (DoR) e Concluído (DoD)

## DoR (Definition of Ready)
- História com objetivo claro (formato "Como/Quero/Para").
- Critérios de aceite (Gherkin) definidos.
- Dependências mapeadas e labels aplicadas.
- Impacto em segurança/LGPD considerado.
- Estimativa inicial e prioridade definidas.
- Para itens de frontend, requisitos de identidade visual HW1 descritos (tokens, componentes e estados).

## DoD (Definition of Done)
- Critérios de aceite aprovados.
- Testes unitários/cenários Gherkin cobrindo fluxo principal.
- Logs/auditoria implementados (quando aplicável).
- Documentação e help contextual atualizados.
- Observabilidade (métricas/erros) instrumentada.
- Interface frontend aderente ao padrão visual HW1 (sem cores hardcoded fora do design system).
- Registro de andamento no board interno e changelog curto (sem exigência de relatório detalhado de entrega externa).
